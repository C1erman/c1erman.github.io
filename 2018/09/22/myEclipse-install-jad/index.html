MyEclipse 10安装反编译插件

发表于 2018-09-22 | 更新于 2019-01-28 | 分类于 -JavaWeb

掌握业务逻辑，弄清实现方式的一个好方法就是查看源代码。

本篇文章仅作记录，如何利用jadclipse插件在myEclipse上查看到已经被编译过的.class文件的源代码。

问题

关于”轮子哥“的事迹及衍生事物，我也有所耳闻。只懂如何使用轮子是否可行，这个问题在暂且不表。

但是，知道轮子的制造方法是很重要的。

”拿来主义“用的久了，有一天手没了怎么办。

回到本篇文章本身上，有需要查看一些jar包的时候，但是直接在myEclipse中是打不开的，这些.class是经过编译的。直接打开，只会看到很多乱码。

解决

下载

jad.exe

net.sf.jadclipse_3.3.0.jar

存放

  • jad.exe

    将此文件放在jdk/bin文件夹下，如若不记得自己正在使用哪一个jdk，右键点击我的电脑，选择高级系统设置，打开环境变量查看jdk文件夹位置。

  • jadclipse.jar

    将此文件放在myEclipse/dropins文件夹下。

配置

  • 设置jadClipse

    打开myEclipse，依次点击

    Windows=>Preferences=>Java=>JadClipse=>Path to decompiler

    在本框中输入jad.exe文件的绝对位置即可。

  • 设置使用jad查看.class文件

    依次点击

    Windows=>General=>Editors=>File Associations

    设置.class文件和.class without source文件的默认（Default）为JadClipse Class File Viewer，如果没有就自己添加。

查看

完成以后，点击自己想了解的.class文件即可看到正确的java代码。

警惕

没有出现JadClipse窗口

按以下文件路径查看bundle.info文件，推荐使用Notepad++。

myEclipse=>configuration=>org.eclipse.equinox.simpleconfigurator=>bundles.info

找到net.sf.jadclipse,3.3.0,dropins/net.sf.jadclipse_3.3.0.jar,4,false一行，看自己这一行的内容是否类似（即能准确定位到jadclipse.jar）。

例如，因为我的jadclipse.jar放在dropins文件夹里，所以上述代码行中dropins/net.sf.jadclipse_3.3.0.jar正确的反应出了jar包的位置。
