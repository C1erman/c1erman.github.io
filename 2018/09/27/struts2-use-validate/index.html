JSP表单验证

发表于 2018-09-27 | 更新于 2018-10-11 | 分类于 -JavaWeb

来一起稍微谈谈关于JSP表单验证的事。

闲言

在互联网技术，或者说服务器脚本语言还不发达的时候，人们使用服务器来验证用户所提交的每一张表单。而这样做所带来的问题是显而易见的：用户在提交表单之后，不仅要等待些许时间才能够得知自己的填写是否出错，而且无故的浪费掉了宝贵的服务器资源。

随着各种脚本语言的出现与不断迭代，服务器逐渐摆脱了“表单验证”这一工作（某些安全验证除外），并将此项工作交付于脚本语言。

以Javascript为例，我们完全有能力在每张表单被提交至处理程序之前，初步验证表单信息是否符合我们的预期，提醒用户使其及时修改，并且还可以顺手做到在一段时间内禁止再次触发submit事件。

多么美妙。

然而，服务器端表单验证依然存在着各种各样的需求，某些数据需要双重验证自不必说，作为服务器页面，使用JSP本身来实现表单验证必不可少。

实现

作为框架而言，整合内容、减少开发负担等，是其基本要求。

Struts2自是如此，它本身可提供各种各样的拦截器，每一张通向处理程序的表单想必非常辛苦。

在诸多”验证器“之中，存在着一个名为validate的家伙，我们可以通过实现该验证器以实现数据验证功能。该验证器的实现有以下两种途径：

  • 实例化Validateable接口

    我们经常使用的ActionSupport类已经以空方法的手段实现了该接口，因此只需要顺便在自己的Action中重写validate()方法即可。

  • 使用XWork验证框架

    考虑到存在众多需要验证的数据类型及类型，使用Struts2提供的validator框架即可轻松完成些许常见的验证需求。

实例化Validateable接口

业务流程

首先大略纵览一张表单的提交所游历过的地点。

  1. 提交，数据类型转换，进入Action
  2. 调用setter()方法赋值
  3. 调用validate()方法进行数据校验
  4. 调用execute()方法以获取result
  5. 进入struts.xml对应result

编写

手动编写自己所需要的东西，往往是最普通却最有效的方法。

继承ActionSupport类，依据个人喜好重写validate()方法即可。

在这个方法中，我们需要通过使用this.addFieldError(String fieldName,String errorMessage)方法，来设置当用户填写数据的格式与预期不符时所抛出的信息。同时，在struts.xml文件中的<action>标签下添加如下标签：<result name="input">表单提交页面</result>，捕获抛出的信息，并携带该信息跳转到原表单提交页面。

通过：<%@ taglib prefix="s" uri="/struts-tags"%>引入struts标签库后，可以使用<s:fielderror />标签，将捕获到的信息（所有信息）输出在原表单登陆页面。

当然，由于我们已经指定了fieldName，完全可以不使用这个标签，异常信息将依照fieldName被输出到正确的位置上。

使用XWork验证框架

当采用上述方法时，最有可能出现的问题就是庞大的validate()方法让程序变得越来越难以维护。

所以，可以使用struts2所提供的validator框架，通过配置xml文件，可以使用到框架所提供的16个表单验证器，如：requiredstring、regex等。

使用

  1. 通过xml定义验证条件

    在表单所提交的Action文件夹下，以命名规则actionClassName-validation.xml新建xml文件，此时对整个action都有效；若以命名规则actionClassName-methodName-validation.xml新建xml文件，此时只对该method有效。

    在此xml文件中，首先声明规则：

    <!DOCTYPE validators PUBLIC

    "-//Apache Struts//XWork Validator 1.0.2//EN"
"http://struts.apache.org/dtds/xwork-validator-1.0.2.dtd">

    在<validate>与</validate>之间定义验证条件即可。

  2. 微调struts.xml

    与之相似，我们同样需要在struts.xml文件中添上：<result name="input">表单提交页面</result>以捕获异常信息。
