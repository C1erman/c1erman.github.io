Git你好

发表于 2019-11-10 | 更新于 2019-11-12 | 分类于 -前端知识

多人协同开发的首选利器！

基本概念

Git是一个分布式版本控制工具。

  • 不存在中央物理服务器，鸡蛋不放在一个篮子里，安全性较强。
  • 本地仓库的存在使得工作时网络环境不再是必须的。

此外，Git将文件处理位置定义为四个区：工作区、暂存区、本地仓库和远程仓库。它们之间的转换关系如下图所示：

基础操作

克隆仓库

使用命令git clone <远程仓库地址>来克隆一个远程仓库。

在执行推送push或者拉取fetch/pull的时候，如果省略了远程主机的名称，则默认使用名为origin的远程数据库。因此一般都会把远程数据库命名为origin。

使用命令git remote获取当前仓库在本地的别名，可使用命令git remote rename old_name new_name修改某个远程仓库在本地的简称。

克隆指定分支，使用命令git clone -b branch_name <远程仓库地址>。

拉取

使用命令git fetch <远程仓库地址> <分支名>拉取远程仓库的某一分支。

使用命令git pull <远程仓库地址> <远程分支名>:<本地分支名>拉取远程分支并于本地分支合并。

推送

使用命令git push <远程仓库地址> <远程分支名>:<本地分支名>将本地分支的更新推送到远程主机。

索引

使用命令git add <文件/路径>命令将文件或文件夹内容添加到索引。

注释

使用命令git commit将更改记录添加到本地仓库，加上-m参数可为提交记录加上一个注释。

分支操作

使用命令git branch列出现处分支，加上-r参数列出远程仓库所有分支。

使用命令git branch branch_name新建一个本地分支。

使用命令git branch -D branch_name删除本地分支。

使用命令git checkout branch_name切换到指定分支，如若该分支不存在，则会自动创建。

应用

创建子分支

以在dev分支下创建dev-c1er分支为例：

  1. 创建分支：在dev分支下git checkout -b dev-c1er。
  2. 定位分支：选择父分支git checkout dev，并拉取父分支最新内容git pull。
  3. 提交子分支：在dev分支下提交内容git add .并创建分支git push --set-upstream origin dev-c1er。
  4. 添加本次注释git commit -m "add new branch dev-c1er"。

如何将本地分支与远程分支相关联：

用git push --set-upstream origin branch_name来在远程创建一个与本地branch_name同名、以当前分支为父分支的新分支并跟踪；用git checkout --track origin/branch_name同名的分支并跟踪分支。

GIT文件冲突

  • 本地修改了一份文件，远端仓库的该文件已经被修改时的pull，保存本地修改。

    通过git stash，将工作区回复到上次提交的内容，同时备份本地做的修改，之后继续执行git pull，成功之后执行git stash pop将之前本地做的修改应用到当前工作区。

  • 本地修改了一份文件，远端仓库的该文件已经被修改时的pull，放弃本地修改。

  • CONFLICT字样的内容冲突：直接编辑冲突文件后提交。

分支更新

将一个关联于主分支的子分支更新到与主分支相同的状态。

该操作主要基于git merge命令，作用为把一个分支中的内容合并到当前分支中。

确保主分支为最新的情况下，git checkout到子分支，之后使用git merge命令进行分支合并，将合并后的分支提交到远程仓库对应的子分支，即可实现分支更新。
