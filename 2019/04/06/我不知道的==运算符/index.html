从数据类型开始

原始数据类型とは

原始数据类型（primitive data type）多数人习惯叫做基本数据类型。

按照MDN的解释：

In JavaScript, a primitive (primitive value, primitive data type) is data that is not an object and has no methods. There are 6 primitive data types: string, number, boolean, null, undefined, symbol (new in ECMAScript 2015).

需要注意的是：原始数据类型不是对象，并且自身不提供方法。

原始数据类型如下所示：

  • Undefined、Null
  • Boolean、String、Number、Symbol

你可能会问：Object呢，它既然不是原始数据类型，那算是什么呢？

我的回答是：Object算是一种数据类型，除了原始数据类型以外的类型都继承自Object，就像Array、Function等。

实际上，我通常把Array这些叫做是一种引用类型。

原始值和对象

那么，原始值（这里指原始数据类型）与对象的区别是什么呢？

两者最主要的区别是：每个对象都有唯一的标识且只（严格地）等于自己，而对原始值只要编码值相同，则被认为相同。

原始值

原始值具有以下特点：

  • 按值进行比较
  • 不可改变：其属性不能被改变、添加、移除

对象

对象具有以下特点：

  • 按引用进行比较
  • 默认可变：对象属性可以很自由的被改变、添加、移除

类型检测

有两种对值进行分类的操作符：typeof、instanceof。

还有一个方法：Object.prototype.toString.call(object)。

typeof主要用于原始值

注意主要这两个字，因为有function的存在。

typeof null的历史

第一代JavaScript引擎的值表示为32位的字符，最低的3位作为值的类型标识，表示该值是对象、浮点数或其他。

对象的标识是000，而使用机器语言NULL指针的null，它的所有位均为0，这样一来，typeof检测值的标识位时，null便表示为object了。

instanceof用于对象

使用==时发生了什么

将值转换为原始值

要将任意值转换为数字或字符串，首先会被转换为任意的原始值，然后再转换为最终的结果。

ECMAScript规范中有一个内部函数，ToPrimitive()，不能被访问到，但是它能帮助你理解对象是如何转换为数字和字符串的。

该函数形如：ToPrimitive(input,PreferredType?)。

可选参数PreferredType表明转换后的类型：它可以是Number或String，具体取决于当时的情况。

如果PreferredType为Number，会执行以下步骤：

  1. 如果input是原始值，则返回input
  2. 如果input是对象，调用input.valueOf()，如果其结果是原始值就返回
  3. 如果结果不是原始值，调用input.toString()，如果其结果是原始值就返回
  4. 如果结果不是原始值，抛出TypeError

而PreferredType为String时，只是更改第2、3步的执行顺序。

因此，一个对象重写了valueOf()或者toString()，就分别会影响Number()或String()。

宽松比较

如果两个运算数的类型相同，是六种规范类型Undefined、Null、Boolean、Number、Object时，使用严格相等比较它们。

如果两个运算数的类型不同，则会触发以下操作：

  • undefined和null被认为是宽松相等的，而不严格相等
  • 一个字符串和一个数字时，将字符串转换为一个数字，再使用严格相等比较两者

  • 一个布尔值和非布尔值，将布尔值转化为0或1，再进行宽松比较

  • 一个对象和一个数字或一个字符串，则将对象转为原始值，再进行宽松比较

  • 其它均为false
