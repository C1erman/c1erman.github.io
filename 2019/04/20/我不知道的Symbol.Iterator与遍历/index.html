我不知道的Symbol.Iterator与遍历

发表于 2019-04-20 | 更新于 2019-04-21 | 分类于 -我不知道的JavaScript

ES6借鉴了C++、Java等语言，引入了for…of循环，作为遍历所有数据结构的统一方法。

要被遍历的数据结构必须部署Symbol.iterator属性，即iterator接口。

for...of可以消费这个接口。

ECMAScript6

ECMAScript 6是JavaScript语言的下一代标准，于2015年6月正式发布，它是一个泛称，涵盖了ES2015、ES2016、ES2017等。

自1996年11月JavaScript被提交到国际标准化组织ECMA、并于1997年发表ECMA-262号标准文件，即浏览器脚本语言的标准ECMAScript 1.0后，已经过去23年了。

ES6带来了一系列新的特性：let命令、const命令实现了块级作用域，字符串支持了Unicode表示，实现了函数的默认参数，箭头函数解决了一部分this的问题，新的基础数据类型Symbol，以及新的数据结构Set与Map，更为重要的是Promise、Generator、async使得原生异步编程成为了可能。

当然，上述提到的仅仅是这些崭新特性的一瞥，如需了解ES6的全部特性，请关注ES6的最新进展、ES6入门、MDN。

强大的Iterator

由于ES6引入了新的数据结构：Map、Set。继续使用for...in或者Array.forEach()已经不太够用了，我们希望用一种统一的方式来遍历不同类型的数据结构，在这种情况下，Iterator应运而生，它有以下几种作用：

  • 为各种数据结构提供统一的访问接口
  • 控制数据结构成员的访问顺序
  • 配合for...of进行遍历

ES6规定：默认的Iterator接口将是某一数据结构的Symbol.iterator属性。换句话说，一个数据结构只要有这个属性，就能够被遍历。

拥有Iterator的

原生具备Iterator接口的数据结构如下所述：

  • String
  • Array
  • 函数的arguments对象
  • NodeList对象
  • Set
  • Map
  • TypedArray

除此之外，其它数据结构，多数指Object，的Iterator接口都需要我们自己去添加。

Object默认没有Iterator：

我们使用Object来作为数据结构使用，主要是因为它是键值对的实现，而ES6已经有了Map，我们就没有必要使用Object了。

对比for...in

想一想，我们之前在哪里使用了for...in呢？

在遍历对象属性时使用了：该方法会遍历某一对象的所有可枚举的属性。

虽然系统创建的属性不可枚举，而用户创建的属性可以枚举。但是还是不建议使用。

for...in有如下缺点：

  • 由于for...in遍历键名，但在遍历数组时，本应为数字的键名却被转换为字符串
  • 会遍历某一对象的原型链上所有可枚举属性
  • 它会以任意顺序遍历键名

for...of的显著优点：

  • 没有for...in的那些缺点
  • 不同于forEach，它可以与break、continue、return一起使用，因为forEach无法中途跳出循环

所以，呃，用键名时用for...in，其它情况还是多使用for...of吧。
