对 webpack4 配置的一些记录。

webpack

webpack4支持0配置打包，默认地，webpack会打包./src下的index.js入口文件，并生成./dist文件夹，其中包含已打包过的main.js文件。

这对我们来说是万万不够的，我们可以通过文件webpack.config.js来配置webpack的行为。

模式

对webpack4来说，mode的取值表明了打包的模式。

  • none

    默认值，无任何压缩、混淆等优化处理。

  • production

    生产模式下，会进行对JavaScript的压缩与混淆，同时也预设了一些其它的plugins。

  • development

    开发模式下，打包出来的内容自然是开发友好的。

    默认使用了webpack.NamedModulesPlugins、webpack.NamedChunksPlugins。前者使每个模块拥有独一的name，后者使每个chunks拥有自定义的name。

导出

关于 output 部分，需要我们注意的是 publicPath 属性，其为项目中的所有静态资源指定一个基础路径。

打包之后的静态资源访问路径可以表示为：path = output.publicPath + 相关loader配置路径。

该属性的默认值为空白字符串，也即 webpack.config.js 文件的当前目录。

模块解析

通过对resovle的设置，我们可以实现对webpack解析模块的规则设置。

常用配置属性解释如下：

  • modules

    设置 webpack 寻找第三方模块的路径，为 Array 类型，默认值为 node_modules 文件夹。考虑到项目中会有一些组件/模块的复用程度较高，为了避免过长的路径名，此时将组件/模块的源文件夹路径写在 modules 中即可。

    本质上只是路径名的替换，并没有魔法。

  • extensions

    导入组件/模块时，我们并没写后缀名。这就是 extensions 的功效，它是一个字符串 Array。

  • alias

    通过该配置属性，使用别名把源导入路径映射成一个新的导入路径。

    以上述代码为例，使用 import Get from 'c1er/Http' 时，将通过别名映射为 import Get from './src/utils/Http'。

babel模块

常用的Babel模块如下所述：

  • @babel/cli：babel搭建的脚手架工具
  • @babel/core：babel核心模块
  • babel/loader：用于编译JavaScript代码的loader
  • @babel/runtime：用于手动引入可复用的垫片
  • @babel/preset-env：将最新JavaScript代码转义为浏览器能支持的版本，就像一个套餐，提供了一系列的解决方案
  • @babel/preset-react：用于编译React的JSX语法
  • @babel/plugin-transform-runtime：搭配@babel/runtime，自动引入可复用的垫片
  • @babel/plugin-proposal-class-properties：解析Class的属性
  • @babel/plugin-proposal-decorators：解析装饰器模式语法
  • @babel/plugin-proposal-export-default-from：解析export~from语法

@符号是Babel7的调整，代表官方的域。

plugin与preset

plugin是preset的集合，如果你仅需要转义某一特定的特性，那么只需要preset即可。

它们的执行顺序如下：

  1. 执行所有plugin，再执行preset
  2. 多个plugin，按照声明次序执行
  3. 多个preset，按照声明次序执行

loader与plugin：

loader用于加载某些资源文件，如css文件、图片或JSX；plugin用于扩展webpack的功能。
