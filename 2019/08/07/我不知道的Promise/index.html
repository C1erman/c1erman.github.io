我不知道的Promise

发表于 2019-08-07 | 更新于 2019-09-13 | 分类于 -我不知道的JavaScript

Promise对象用于表示一个异步操作的最终状态。

但我更喜欢这种表述：Promise对象用于代理一个值，这个值在Promise对象创建时可能是未知的，Promise允许你为异步操作的成功或失败添加对应的handler以处理返回结果。

构造函数

Promise构造函数执行时立即调用executor函数，之后才是new操作符对Promise()的调用。

在executor函数中，你可以进行一些异步操作，并依据你想要的结果，分别调用resolve或reject函数。

这两个函数是JavaScript引擎提供的，你可以将这两个函数视为值的传递者，事实上也同样如此：使用resolve(value)将Promise对象的状态置为fulfilled、使用reject(value)将Promise对象的状态置为rejected，传递的值将交由then()进行捕获。

Promise对象的状态：

使用Promises/A+社区规范的状态标识。

Fulfilled：成功。

Rejected：失败。

Pending：既不是Fulfilled也不是Rejected，初始化状态。

注意：

Promise确保了其状态的改变只有一次，FulFilled状态与Rejected状态均可称为Settled状态。

事件回调函数

将Promise原型对象提供的then()、catch()与finally()视为事件回调函数也没什么不妥：它们会依据Promise对象的状态而被分别调用。

但是需要注意的是：这些回调函数会在本轮eventLoop执行完之后、下轮eventLoop执行之前被被调用，准确的来说，这些任务被添加在一个微任务队列中。

then

  • 函数签名
  • 返回值

该函数返回一个新的Promise对象，该Promise对象的状态表述如下：

then()正常执行时，返回一个携带值为then()的返回值或undefined的、fulfilled的Promise对象。

then()执行过程中抛出异常时，返回一个携带值为异常信息的、rejected的Promise对象。

then()返回Promise对象时，返回的Promise对象的状态同前述对象保持一致，携带值也保持一致，行为也保持一致。

携带值：指该Promise对象的接收/拒绝状态的回调函数的参数值。

  • 介绍

当Promise的状态为fulfilled时，将会调用onFulfilled(value)；为rejected时将会调用onRejected(reason)。

如果传入onFulfilled参数的类型不是函数，则会被替换为x => x，即原样返回传入的值。

如果没有返回值，下一个回调函数将无法获取上一个Promise的结果：

finally

  • 函数签名
  • 返回值

一个设置了finally回调函数的新的Promise对象，该Promise对象的状态任意。

注意：在finally()函数体中throw异常时，将返回一个携带值为throw原因、状态为rejected的Promise对象。

  • 介绍

就像try~catch~finally一样，如果你需要在Promise调用链结束时执行一些指定的代码，而无论Promise对象的状态如何时，使用该方法将会是一个好选择。

  • 注意

虽然finally()与.then(onfinally, onfinally)类似，但仍需要注意：

finally()调用内联函数时，不需要多次声明该函数或为该函数创建一个变量保存它。

由于无法知道Promise对象的最终状态，所以finally()回调函数不含有任何参数。

未显式返回Promise对象或抛出异常时，finally()会保存上一层Promise对象的状态与信息。
