我不知道的Function

发表于 2019-08-26 | 更新于 2019-09-19 | 分类于 -我不知道的JavaScript

函数是JavaScript中的第一类公民。

闭包或许是你深入了解函数的第一步，但除此之外，还有一些你需要了解的内容。

声明变量的一种方式

function关键字是JavaScript中声明变量的一种方式，直至ECMAScript 6，声明变量的方式一共有六种：var、let、const、function、class和import。

从实参到形参

函数的参数是按值传递的，就像将一个变量赋值给另一个变量。

补充：对象的属性赋值也是按照这种形式。

正如我们了解的那样：变量是名字到内存空间的映射，显然，对存放原始数据类型的变量来说，这块内存空间存的即是值；而对存放对象的变量来说，这块内存空间存的是对象在堆中的引用。

而变量之间的赋值过程就显而易见，拷贝一个值或者拷贝一个引用。

arguments对象

arguments对象是所有非箭头函数中都可以使用的局部变量，正如你所知道的，它是一个类数组对象。

该对象有数个值得关心的属性：

  • arguments.callee

指向当前正在执行的函数。

在大多数情况下，我们会在匿名函数的递归调用中用到这一属性：

在ES5的严格模式下，该属性被禁止。

  • arguments.caller

指向调用当前函数的函数。该属性已被废弃

尽管该属性已被废弃，但你仍然可以使用Function.caller，换句话说是arguments.callee.caller，得到相同的指向。

  • arguments.length

代表传递给当前函数的参数数量。

这一数量可能比函数的形参大，也可能比函数的形参小。

注意：rest参数会影响该值

函数式编程

正如在本文开始时提及的那样：函数是JavaScript的第一类公民，这意味着JavaScript可以进行函数式编程。

声明式编程与命令式编程

函数式编程是更广义编程范式的一部分：声明式编程。

声明式编程的一个特点就是，对执行结果的描述远胜于执行过程。为了加深印象，我们将声明式编程与命令式编程做对比，命令式编程的特点为，重点关注于实现目标的具体过程。

何为函数式编程

简单的讲，我们能使用函数就像使用变量一样。

这对JavaScript使用者可能是理所当然的，因为在JavaScript中函数就是变量。

  • 像使用变量一样，把函数做为其它函数的参数
  • 像使用变量一样，作为函数的返回结果

核心：不可变性

在函数式编程中，输入的数据是不可变的，即函数不能有副作用。

即使我们需要对输入的数据进行操作，那也是对原生数据的拷贝上进行操作，而不影响原生数据。

纯函数

纯函数是一个返回结果只依赖于输入参数的函数，它至少有一个参数，并且总是返回一个值或一个函数。

纯函数体现着不可变性。

回调函数

事实上，回调函数是从函数式编程中衍生出来的概念。这相当容易理解：回调函数即是将一个函数作为参数传递给另一个函数。
