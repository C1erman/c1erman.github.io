Flex布局规范翻译

发表于 2019-10-03 | 更新于 2019-10-05 | 分类于 -前端知识

本文简单翻译Flex布局规范中的部分章节。

章二：Flex布局盒模型和术语

Flex容器是由元素生成的盒子，该元素的display属性被计算为flex或inline-flex。Flex容器的流内子元素被称为Flex项目，并使用Flex布局模型进行布局。

与块和内联布局不同：其布局计算偏向于块和内联流向，而Flex布局偏向于Flex的方向。为了更轻松的讨论Flex布局，本节定义了一组与Flex Flow相关的术语。Flex Flow的值与书写方式确定这些术语如何映射到物理方向：顶部/右侧/底部/左侧，轴：水平/垂直，尺寸：宽度/高度。

  • 主轴：main axis、主维度

    可伸缩容器的主轴是Flex项目沿其排列的轴线，它在主维度上延伸。

  • 主起点：main-start、主终点：main-end

    将Flex项目从主起点开始放置在容器中，朝着主终点方向延伸

  • 主尺寸：main size、主尺寸属性

    Flex容器或Flex项目的宽度或高度，以主维度为基准，是该盒子的主尺寸。因此，其主尺寸属性是width或height属性，无论是否在主维度上。类似的，其最小和最大尺寸属性是其min-width/max-width或min-height/max-height，以主维度为基准，并确定其最小/最大主尺寸。

  • 交叉轴：cross axis、交叉维度

    垂直于主轴的轴称为交叉轴。它在交叉维度延伸。

  • 交叉起点：cross-start、交叉终点：cross-end

    Flex线被Flex项目填充，并且沿着交叉起点向着交叉终点被放置。

  • 交叉尺寸：cross size、交叉尺寸属性

    Flex容器或Flex项目的宽度或高度，以交叉维度为基准，是该盒子的交叉尺寸。因此，其交叉尺寸属性是其width或height属性，无论是否在交叉维度中。同样，其最小和最大交叉尺寸属性是其min-width/max-width或min-height/max-height属性，以交叉维度为基准，并确定其最小/最大交叉尺寸。

章三：Flex容器：flex和inline-flexdisplay值

名称 display
新值 flex｜inline-flex

  • flex

    当放置在流布局中时，此值使元素生成一个块级的Flex容器。

  • inline-flex

    当放置在流布局中时，此值使元素生成一个内联级别的Flex容器。

一个Flex容器为其内容建立一个新的Flex格式化上下文。除了使用Flex布局而不是Block布局外，这与建立块格式化上下文相同。例如，浮动元素不会侵入Flex容器，并且Flex容器的margin不会与其内容的margin折叠。Flex容器就像块级容器一样，为其内容建立了一个包含块。overflow属性适用于Flex容器。

Flex容器不是块级容器，因此某些以块布局为前提设计的属性不适用于Flex布局。尤其是：

  1. float和clear不会为Flex项目产生浮动或间隙，也不会使其流出。
  2. vertical-align属性对Flex项目没有影响。
  3. ::first-line和::first-letter伪元素不适用于Flex容器，并且Flex容器不向其祖先提供格式化第一行或第一字母。

如果元素的display属性被显式设置为inline-flex，则在某些情况下其display属性被计算为flex：CSS 2.1/CSS 2.2第9.7节中的表格被修改为包含附加行，其中inline-flex位于指定值列，而flex位于计算值列。

章五：排列和方向

Flex容器的内容可以按任意方向和顺序排列。这使得开发人员可以轻松实现以前需要复杂或不稳定方法达到的效果，例如使用float和clear属性进行hack。排列和方向通过属性flex-direction、Flex-wrap和order属性实现。

flex-direction属性通过设置Flex容器主轴的方向来指定Flex项目在Flex容器中的放置方式。这确定了Flex项目的布局方向。

  • row

    Flex容器的主轴与当前书写模式的嵌入式轴具有相同的方向。主起点方向和主终点方向分别等于当前写入模式的行内开始方向和行内结束方向。

  • row-reverse

    与row相同，只是交换了主起点和终点的方向。

  • column

    Flex容器的主轴与当前书写模式的块轴方向相同。主起点方向和主终点方向分别等于当前写入模式的块起点方向和块终点方向。

  • column-reverse

    与cloumn相同，只是交换了主起点和终点的方向。

注意：反向值不会反转盒顺序：像writing-mode和direction一样，它们仅改变流向。绘制顺序、播报顺序和顺序导航顺序不受影响。

order属性通过将Flex项目分配给顺序组来控制其在Flex容器中的显示顺序。它需要一个整数值，该值指定Flex项目所属的顺序组。

Flex容器按顺序修改后的文档顺序排列其内容，从文档编号最低的顺序组开始，然后向上。具有相同顺序组的项目按它们在源文档中出现的顺序进行排列。这也影响绘制顺序，就像在源文档中重新排列了Flex项目一样。将Flex容器的绝对定位子元素视为具体属性order : 0以确定它们相对于Flex项目的绘制顺序。

章七：伸缩性

Flex布局在定义方面是使Flex项目Flex的能力，可以更改其width/height以填充主维度中的可用空间。这是通过flex属性完成的。Flex容器将自由空间分配给其Flex项目（与其flex-grow属性成比例）以填充容器，或者收缩它们（与其flex-shrink属性成比例）以防止溢出。

如果Flex项目的flex-grow与flex-shrink的值均为零，则它是完全不变形的，否则为具有伸缩性的。

flex属性指定伸缩长度的组成部分：伸缩系数（增长grow与收缩shrink）和伸缩基准basis。当盒子是Flex项目时，将使用flex而不是main size属性来确定盒子的主尺寸。如果盒子不是Flex项目，则flex属性无效。

  • flex-grow

    这个使用数字取值的组成部分是flex-grow的完整写法，它指定了伸缩增长因子。该因子决定了正在分配正的自由空间时，Flex项目相对于Flex容器中的其余Flex项目将增长多少。省略时，它将被设置为1。

    0~1之间的flex-grow值有一些特殊的行为：当行上的flex-grow值的总和小于1时，它们将占用少于100%的可用空间。

    如果当前行只有一个Flex项目，则会默认填充整行。

  • flex-shrink

    这个使用数字取值的组成部分是flex-shrink的完整写法，它指定了伸缩收缩因子。该因子决定了正在分配负的自由空间时，Flex项目相对于Flex容器中的其余Flex项目将收缩多少。省略时，它将被设置为1。

    注意：在分配负的自由空间时，伸缩收缩因子乘以伸缩基准。这会与Flex项目能够收缩的程度成正比地分配负空间。例如，便于在大尺寸Flex项目明显变小之前，小尺寸Flex项目不会缩小为零。

  • flex-basis

    flex-basis接收与width/height属性相同的值（除了对auto的区别对待）以及content关键字：

    • auto

      在Flex项目上指定时，auto关键字将检索main size属性的值作为使用的flex-basis。如果main size的值本身是auto的，则使用的flex-basis则为content。

    • content

      指定基于Flex项目内容的自动尺寸。（它通常等效于最大内容大小，但要进行内容调整以处理纵横比、固有大小限制和直角流；关于这一点，请参阅章九：Flex布局中的算法详情。）

      注意：此值在Flexible Box Layout的初版中不存在，因此某些较早的实现不支持该值。通过将主尺寸（width/height）设置为auto可以达到相同的效果。

    • width

      对于所有其它值，flex-basis的解析方式与width/height属性相同。

    当flex-basis属性从flex速记属性中省略时，其指定值为0。

  • none

    关键字none将被扩展为0 0 auto。

上图显示了绝对增长（以0为基准）和相对增长（以Flex项目内容的大小为基准）之间的差异。这三个Flex项目的伸缩增长因子分别为1、1、2；请注意，伸缩增长因子为2的Flex项目的自由空间占比为其它Flex项目的两倍。

flex属性的初始值为0 1 auto。

注意：在flex速记属性中省略时，flex-grow和flex-basis的初始值与默认值不同。这样，flex速记属性可以更好的适应最常见的情况。

必须将尚未包含两个增长/收缩因子的无单位0解释为增长/收缩因子。为避免误解或无效的声明，开发者必须为值为0的flex-basis部分指定一个单位，或在其前面加上两个增长/收缩因子。

flex属性的基础值

下面的列表总结了代表最常见效果的四个flex属性的取值：

  • initial

    等效于flex : 0 1 auto（就是flex属性的初始值）。根据Flex项目的width/height属性调整项目的大小。（如果该Flex项目的main size属性的计算值为auto，则将根据其内容调整Flex项目的大小。）当存在正的自由空间时使Flex项目无法增长，但在空间不足时允许其缩小至最小尺寸。对齐功能或者取值为auto的margin属性可用于沿主轴对齐Flex项目。

  • auto

    等效于flex : 1 1 auto。根据Flex项目的width/height属性调整项目的大小，但是使其拥有极其充分的灵活性，以便它们吸收主轴上的任何自由空间。如果所有Flex项目均为flex : auto、flex : initial或flex : none，则在调整大小后将任何正的可用空间平均分配给具有flex : auto的项目。

  • none

    等效于flex : 0 0 auto。此值根据Flex项目的width/height属性调整项目的大小，但是使Flex项目完全不伸缩。这类似于初始值，但是此值即使在溢出情况下也不允许收缩Flex项目。

  • 正数

    等效于flex : 正数 1 0。使Flex项目具有伸缩性，并将伸缩基准设置为0，从而使该Flex项目接收Flex容器中指定比例的自由空间。如果Flex容器中的所有项目都使用此模式，则它们的大小将与指定的flex正数成比例。

默认情况下，Flex项目不会缩小到其最小内容大小（最长的单词或固定大小的元素的长度）以下。要更改此设置，请设置min-width或min-height属性。（请参阅章四第五节：弹性项目的自动最小大小。）

flex-basis属性设置伸缩基准。它接收与width和height属性相同的值，以及content关键字。

对于auto与content以外的值，flex-basis的解析方式与水平书写模式中的width属性相同，不同之处在于，如果一个值对width属性来说解析为auto，它将对flex-basis属性解析为content。例如，根据Flex项目的包含块（即其Flex容器）解析得到的flex-basis的百分比值；并且如果包含块的大小不确定，则flex-basis的使用值为content。另外一个必然结果是，除非另有规定（例如通过box-sizing属性进行指定）否则flex-basis会确定大小为content box。

章八：对齐

在Flex容器的内容完成其伸缩并确定所有Flex项目的尺寸之后，即可在Flex容器中对齐它们。

margin属性可以用于在类似于块级布局对齐项目那样以强大的方式来对齐Flex项目。Flex项目还遵循CSS Box Aliginment中的对齐属性，该属性允许在主轴和交叉轴上轻松进行基于关键字的项目对齐。这些属性使对齐的许多普通类型变得微不足道，包括CSS 2.1中非常困难的某些事情，例如水平居中和垂直居中。

注意：虽然对齐属性是在CSS Box Alignment中定义的，但Flexible Box Layout在此处重现了相关属性的定义，以免形成规范性的依赖关系，而这可能减缓规范的发展。这些属性仅适用于Flex布局，直到CSS Box Alignment Level 3完成并定义其在其它布局模式下的效果。此外，Box Alignment模式下定义的任何新值都将应用于Flexible Box Layout；换句话说，Box Alignment模块一旦完成，将取代此处的定义。

使用自动margin进行对齐

本节是非规范性的。“边距如何影响布局”的标准定义在Flex Layout算法部分。

Flex项目的auto取值margin属性效果与块流中的自动margin效果非常相似：

  • 计算伸缩基准和伸缩长度时，自动margin被视为0。
  • 在通过justify-content和align-self对齐之前，任何正的自由空间都会分配给改维度中的自动margin。
  • 溢出盒子会忽略其自动margin，并在最终方向上溢出。

注意：如果将自由空间分配给自动margin，则对齐属性在该维度上将不起作用，因为在伸缩之后margin会偷走所有剩余的可用空间。

flex-content属性将Flex项目沿着Flex容器的当前行的主轴对齐。在确定了任何伸缩长度和任何自动margin之后，便会进行此操作。通常，当一行中的所有Flex项目都无法伸缩或已达到最大大小时，该属性可以帮助分配剩余的自由空间。当Flex项目溢出行时，它还可以对Flex项目的对齐方式施加一些控制。

  • flex-start

    Flex项目被包装到当前行的开头。将该行上的第一个Flex项目的主起点边缘与该行的主起点边缘紧密放置，并将后续的每个Flex项目与前一个Flex项目紧密放置。

  • flex-end

    Flex项目包装在行尾。将最后一个Flex项目的主终点边缘与该行的主终点边缘紧密放置，并将先前的每个Flex项目与后一个Flex项目紧密放置。

  • center

    Flex项目被包装在该行的中心。行上的Flex项目彼此紧密放置，并在当前行居中对齐，在当前行的主起点边缘和行上的第一个Flex项目之间的间距，与在当前行的主终点边缘与行上最后一个Flex项目之间的间距相同。（如果剩余的自由空间为负，则Flex项目将在当前行的两个方向上均匀的溢出。）

  • space-between

    Flex项目均匀的分布在当前行中。如果剩余的自由空间为负，或者该行上只有一个Flex项目，则则此值与flex-start相同。否则，将该行上的第一个Flex项目的主起点边缘与该行的主起点边缘对齐，将该行的最后一个Flex项目的主终点边缘与该行的主终点边缘对齐。当前行上的剩余Flex项目被均匀分布以使任意两个相邻Flex项目之间的间距相同。

  • space-around

    Flex项目均匀的分布在当前行中，两端均留有（Flex项目之间间距）一半大小的空间。如果剩余的自由空间为负或在当前行中只有一个Flex项目，则此值等于center。否则，将分配该行上的Flex项目，以使该行上的任何两个相邻Flex项目之间的间距相同，并且第一个/最后一个Flex项目与Flex容器边缘之间的间距为两个Flex项目之间间距的一半。

Flex项目可以在Flex容器的当前行的交叉轴对齐，类似于justify-content属性，但是是在垂直方向上对齐。align-items设置所有Flex容器的Flex项目（包括匿名Flex项目）的默认对齐方式。而align-self属性允许针对单个Flex项目覆盖此默认对齐方式。（对于匿名Flex项目，align-self的值始终与其关联的的Flex容器的align-items的值匹配。）

如果任一Flex项目的交叉轴margin，即flex-direction值为row情况下的margin-top/margin-bottom，为auto的话，align-self无效。

值具有以下含义：

  • auto

    将交叉轴对齐的控制交还给Flex容器的align-items值来控制。（这是align-self的初始值。）

  • flex-start

    Flex项目的交叉起点边缘与当前行的交叉起点边缘紧密放置。

  • flex-end

    Flex项目的交叉终点边缘与当前行的交叉终点边缘紧密放置。

  • center

    Flex项目的margin box位于当前行的交叉轴中心。（如果伸缩行的交叉尺寸小于Flex项目的交叉尺寸，则该Flex项目将在两个方向上均等的溢出。）

  • baseline

    Flex项目参与基线对齐：该行上所有的Flex项目将会按照该行的文字基线进行对齐，并且其基线和其交叉起点边缘之间的间距最大的Flex项目，将与该项目的交叉起点边缘紧密放置。如果Flex项目在必要的轴上没有基线，那么将从Flex项目的border box中合成一个基线。

  • stretch

    如果Flex项目的交叉尺寸属性计算为auto，并且交叉轴margin都不是auto的，则Flex项目将被拉伸。它的使用值是使Flex项目的margin box的交叉尺寸尽可能的接近该行的交叉尺寸所需的长度，同时仍要遵守min-height/min-width/max-height/max-width限制。

    注意：如果Flex容器的height属性收到限制，则此值可能会导致Flex项目的内容溢出。

    Flex项目的交叉起点边缘与当前行的交叉起点紧密放置。

当交叉轴上有多余的空间时，align-content属性将Flex容器的行在Flex容器中对齐，这类似与justify-content在主轴中对齐单个项目的方式。注意：此属性对单行Flex容器无效。

值具有以下含义：

  • flex-start

    行被打包到Flex容器的开始。将Flex容器中的第一行的交叉起点边缘放置在与Flex容器的交叉起点边缘平齐的位置，并将随后的每行都放置在与前一行接邻的位置。

  • flex-end

    行被打包到Flex容器的末尾。最后一行的交叉终点边缘与Flex容器的交叉终点边缘齐平放置，并且前一行与后一行接邻放置。

  • center

    行朝向Flex容器的中心打包。Flex容器中的行彼此接邻放置，并在Flex容器的中心对齐，Flex容器的交叉起点与Flex容器的第一行之间的间距，与Flex容器的交叉终点与Flex容器的最后一行之间的间距相等。（如果剩余的自由空间为负，则行在两个方向上均等的溢出。）

  • space-between

    行均匀地分布在Flex容器中。如果剩余的自由空间为负，或者在Flex容器中只有一行，则此值与flex-start相同。否则，将Flex容器中的第一行的交叉起点边缘与Flex容器的交叉起点边缘齐平放置，将Flex容器的最后一行的交叉终点边缘与Flex容器的交叉终点边缘齐平放置，Flex容器中的其它行将被排列以使任意两行之间的间距相等。

  • space-around

    行均匀地分布在Flex容器中，两端各占有（伸缩行之间间距）一半空间。如果剩余的自由空间为负，则此值等于center。否则，将Flex容器中的行分布以使任意两条行之间的间距相同。并且第一行/最后一行与Flex容器边缘之间的间距为伸缩行之间间距的一半。

  • stretch

    行拉伸以占据自由空间。如果剩余的自由空间为负，则此值与flex-start相同。否则，自由空间会在所有行之间平均分配，从而增加其交叉尺寸。

注意：只有在多行Flex容器才能在交叉轴上留出空间已对其伸缩行，因为在单行Flex容器中，唯一的行会自动拉伸以填充空间（指align-items的值默认为stretch）。
