XHR请求与Fetch请求

发表于 2019-10-14 | 更新于 2019-12-25 | 分类于 -前端知识

网络请求。

XHR对象

请求发送流程

  • 初始化一个请求以备发送：open()

    函数签名为：xhrReq.open(method, url, ?async, ?user, ?password)。

    • method

      表示要使用的HTTP方法。对于非HTTP/HTTPS的URL，该方法将被忽略。

    • url

      一个DOMString字符串，表示要发送请求的URL。

      DOMString：是一个UTF-16字符串，对于JavaScript来说，其将直接被映射为String类型。null通常被序列化为"null"。

    • async可选

      布尔参数，默认为true，表示要不要异步发送请求。

    • user可选

      用于认证用途的用户名，默认为null。

    • password可选

      用于认证用途的密码，默认为null。

  • 设置合适的请求头信息：setRequestHeader()

    函数签名：xhrReq.setRequestHeader(header, value)。

    • header

      头部字段名。

    • value

      对应的值。

    此方法必须在open()和send()之间调用。如果多次对同一个请求头赋值，只会生成一个合并了多个值的请求头。

    Accept字段的默认值为*/*。

  • 注册处理函数：onreadystatechange = callback

    当readyState的值发生变化时，会调用callback函数。

    当一个XHR请求被abort()方法取消时，其对应的readystatechange事件不会被触发。

  • 发送特定的请求：send()

    函数签名：xhrReq.send(?data)。

    • data

      可选的请求携带数据，当不发送数据时，使用null。

      可以发送的数据类型为：ArrayBuffer、ArrayBufferView、Blob、Document、DOMString、FormData。

常用属性

  • XMLHttpRequest.readyState只读

    返回一个无符号短整型数字，代表请求所处的状态：

    | 值 | 状态 | 描述 |
    | —- | —————- | ——————————————– |
    | 0 | UNSENT | 请求被创建，但尚未调用open()方法 |
    | 1 | OPENED | open()方法已被调用 |
    | 2 | HEADERS_RECEIVED | send()方法已被调用，并且头部和状态以可获得 |
    | 3 | LOADING | 下载中。responseText属性已经包含部分数据 |
    | 4 | DONE | 下载操作已完成 |

  • XMLHttpRequest.status只读

    返回一个无符号短整型数字，代表请求的响应状态。

    其值为标准的HTTP状态码。在请求完成前，或XHR请求失败时，该值为0。

  • XMLHttpRequest.response只读

    返回一个ArrayBuffer、Blob、Document或DOMString，具体是哪种类型取决于 XMLHttpRequest.responseType的值。

    此值包含整个响应体。

  • XMLHttpRequest.responseType

    该属性是一个枚举类型的属性，表示返回响应数据的类型。我们可以手动设置返回数据的类型，如果将其设置为一个空字符串，它将使用默认的text类型。

    需要确保服务器返回的类型与手动设置的返回类型是兼容的，否则将无法取得返回的数据。另外，无法给同步请求设置该属性。

    常用的取值如下：

    | 值 | 描述 |
    | ————— | ———————————————————— |
    | "" | 即"text"，默认值，实际上为DOMString。 |
    | "arraybuffer" | response是一个包含二进制数据的ArrayBuffer。 |
    | "document" | response是一个HTMLDocument或XMLXMLDocument。 |
    | "json" | response是一个将返回数据视为JSON解析后的JavaScript对象。 |
    | "text" | response是包含在DOMString对象中的文本。 |

  • XMLHttpRequest.withCredentials

    该属性为布尔类型，指示了是否在跨域请求中携带cookies、authorization headers。

    此外，该属性也会被用作响应中cookies被忽视的标记。

    其默认值为false。

Fetch

XHR请求是基于事件回调的异步处理模型，这与Promise大行其道的现在显得格格不入，而Fetch的出现，就是为了解决这个问题。

发送fetch请求

通过函数fetch()发送fetch请求。

其函数签名为：fetch(input, ?init)。

  • input

    定义要获取的资源。可能为：

    • 一个USVString 字符串，包含要获取资源的URL。

      管它那么多，反正都是字符串。

    • 一个Request对象。

  • init可选

    一个配置项对象，包括所有对请求的设置，如下所示：

    • method：请求使用的方法。
    • headers：请求的首部信息，形式为Headers的对象或包含ByteString值的对象字面量。
    • body：请求体信息，可能是一个Blob、FormData对象。注意GET或HEAD方法的请求不能包含body信息。
    • mode：请求的模式，可选值为'cors'、'no-cors'或者'same-origin'。
    • credentials：请求的证书，可选值为'omit'、'same-origin'或者'include'。为了在当前域名内自动发送cookie，必须提供这个选项。
    • cache：请求的cache模式，可选值为'defalut'、'no-store'、'reload'、'no-cache'、'force-cache'或者'only-if-cache'
    • redirect：可用的redirect模式，可选值为自动重定向：'follow'、如果产生重定向将自动终止并且抛出一个错误：'error'、手动处理重定向：'manual'。
    • referrer：一个USVString，可选值为'no-referrer'、'client'或者一个URL。
    • referrerPolicy：指定HTTP首部referer字段的值。可选值为'no-referrer'、'no-referrer-when-downgrade'、'origin'、'origin-when-cross-origin'和'unsafe-url'。
    • integrity：包括请求的子资源完整性值。

该函数的返回值是一个Promise对象，当其resolve时将回传Response对象。

Headers对象

该接口允许你对HTTP请求头和响应头执行各种操作，包括检索、设置、添加和删除。一个Headers对象具有关联的列表，由零个或多个键值对构成。

Headers对象部署了iterator接口，因此可以通过for...of来进行遍历。需要注意的是，for...of Headers ≡ for...of Headers.entries()。

构造函数以及常用的方法如下：

  • Headers()

    构造函数，创建一个新的Headers对象。

  • Headers.append()

    给现有的header添加一个值，或者添加一个未存在的header并赋值。

  • Headers.delete()

    从Headers对象中删除指定header。

  • Headers.entries()

    以迭代器的形式返回Headers对象中所有的键值对。

  • Headers.get()

    以ByteString的形式从Headers对象中返回指定header的全部值。

  • Headers.has()

    以布尔值的形式从Headers对象中返回是否存在指定的header。

  • Headers.keys()

    以迭代器的形式返回Headers对象中所有的header名。

  • Headers.set()

    替换现有的header的值，或者添加一个未存在的header并赋值。

  • Headers.values()

    以迭代器的形式返回Headers对象中所有的header值。

Body混合

Fetch的API提供了Body这个混合（Mixin）代表响应、请求的正文，其被Request与Response实现。

属性

均为只读属性。

  • Body.body

    返回ReadableStream类型的主体内容。

  • Body.bodyUsed

    布尔值，指代Body.body是否被读取。

方法

下述方法均是内容读取操作，会将bodyUsed状态改为true。

  • Body.arrayBuffer()

    Response将挂起一个流操作，返回一个Promise对象，其resovle时的参数数据类型为ArrayBuffer。

  • Body.blob()

    Response将挂起一个流操作，返回一个Promise对象，其resovle时的参数数据类型为Blob。

  • Body.formData()

    Response将挂起一个流操作，返回一个Promise对象，其resovle时的参数数据类型为FormData。

  • Body.json()

    Response将挂起一个流操作，返回一个Promise对象，其resovle时的参数为JSON解析值。

  • Body.text()

    Response将挂起一个流操作，返回一个Promise对象，其resovle时的参数为USVString文本。
