Unity备考知识点

发表于 2019-05-15 | 更新于 2019-05-21 | 分类于 -备考

嗯，选了这门课就要好好准备考试啊。

第一章：Unity简介

特点

Unity引擎的最主要特点是：一次开发就可以部署到所有主流的游戏平台。

面板与视图

  • Hierarchy：层级

Hierarchy视图用于显示当前游戏场景中的每个游戏对象。

  • Inspector：检视

Inspector视图用于显示游戏场景中，当前所选择游戏对象的详细信息和属性设置。包括对象的名称、标签、位置坐标、旋转角度、缩放、组件等信息。

  • Animator：动画控制器

Animator视图可以预览和设置角色行为。

  • Navigation：导航寻路

Navigation视图用于设置游戏角色的自动寻路。

第九章：游戏对象、组件和Prefabs

访问游戏对象

  • 通过名称来查找

使用函数GameObject.Find(str name)，如果场景中存在制定名称的游戏对象，那么返回该对象的引用，否则返回null。

如果有多个重名的对象，则返回第一个对象。

  • 通过标签查找

使用函数GameObject.FindWithTag(str tag)或GameObject.FindGameObjectsWithTag(str tag)来获取指定标签游戏对象的第一个或全部，否则返回null。

组件

任何游戏对象都由组件构成，组件是实现一切功能所必需的，不同的组件有不同的功能。

常用的组件如下：

  • Mesh组件

Mesh：网格组件，如果不为网格添加一个Mesh Renderer组件，则该网格不会被渲染出来。

  • Physics组件

Physics：物理组件，用于模拟真实的物理行为。

  • Scripts组件

Scripts：脚本组件是必不可少的组成部分，它用于添加到游戏对象身上以实现各种交互操纵或其它功能。

组件的访问

  • 对于系统内置的常用组件，只需要直接在脚本里直接访问游戏对象对应的成员变量即可。
组件名称 变量名 组件作用
Transform transform 对象位置、旋转、缩放
Rigidbody rigidbody 物理引擎的刚体属性
Renderer renderer 渲染物体模型
  • 如果访问的不是常用组件，或者欲访问脚本组件：

使用GetComponent<name>()或GetComponents<name>()得到对象的组件或组件列表。

调用该类函数比较耗时，应放在start()中。

Prefabs：预设体

  • 定义

一个游戏对象及其组件的集合与抽象。

  • 优点

使游戏对象及资源能够被重复利用，节约资源、提高开发的效率。

  • 缺点

对预设体进行更改，实例也将同步修改。

预设体的实例化

可通过静态、动态两种方式完成预设体的实例化。

  • 静态

将预设体拖动到Scene视图，或者Hierarchy视图中，便完成了一个预设体的实例化。

  • 动态

即通过脚本的方式进行预设体实例化，核心函数Instantiate有5个重载，常用的函数签名为Instantiate(Object original, Vector3 position, Quaternion rotation)。

函数的三个参数分别为：用来做复制的源对象original、实例对象的位置坐标position、实例对象的四元数旋转坐标rotation。

第十一章：动画系统

Humanoid

Humanoid：人形，通过骨骼映射，去识别其它动画，使之能应用到本身模型上。

  • 可以使用动画重定向功能：即一个人物模型的动画，可以供多个共同骨骼的模型使用。

  • 只通过动画便可以控制人物模型运动。

Avatar

使用Avatar来实现动画的重定向，即动画的复用。

在Rig标签页选择Humanoid后，Avatar Definition常用两种选项

  • Create From This Model：由本模型创建一个Avatar，供自己与其它相同Humanoid的骨骼来共同使用。

  • Copy From Other Avatar：使用其它模型的Avatar。

第十二章：物理系统

Rigidbody：刚体组件

为游戏对象添加Rigidbody组件，实现该对象在场景当中的物理交互。

Rigidbody组件的常用参数如下：

  • Mass：用于设置游戏对象的质量。
  • Drag：设置游戏对象受力运动时的空气阻力。
  • Use Gravity：游戏对象是否使用重力。

Collider：碰撞体组件

选中一个游戏对象，为其添加不同类型的physics组件。

碰撞体组件的常用参数如下：

  • Is Trigger：触发器，选中该项则该碰撞体会忽略物理碰撞，用于触发碰撞事件。

第十四章：光照

光源的类型可分为：

  • Spot：聚光灯

第十八章：坐标系

坐标系

笛卡尔坐标系分为左手坐标系和右手坐标系，左手坐标系是指Y轴指向上方，X轴指向右方，而Z轴指向前方（垂直向里），右手坐标系Z轴则相反。

Unity采用左手坐标系，以X、Y、Z的顺序表示一个位置。

全局坐标系

全局坐标系是用于描述游戏场景内所有物体位置和方向的基准，也称为世界坐标系。

在Unity场景中创建的物体都是以全局坐标系中的坐标原点(0, 0, 0)来确定各自位置的。

局部坐标系

每个物体都有其独立的局部坐标系，它随物体进行相同的移动或旋转。

如果将物体B拖到物体A身上，则物体A就是物体B的父物体，子物体B会以父物体A的坐标点为坐标原点。

视口坐标系

视口坐标已经被归一化，并且它是相对相机的。相机的左下角是(0,0)，右上角是(1,1)，Z轴则取决于相机的位置。

Viewport space is normalized and relative to the camera. The bottom-left of the camera is
(0,0); the top-right is (1,1). The z position is in world units from the camera.

屏幕坐标系

建立在屏幕（游戏运行载体的屏幕）上的二维坐标系，用以描述像素在屏幕上的位置。

坐标转换

  • Transform.TransformPoint(Vector3 position)

将坐标点从局部坐标系转换到全局坐标系。

  • Transform.InverseTransfomPoint(Vector3 position)

将坐标点从全局坐标系转换到局部坐标系。

  • Transform.TransformDirection(Vector3 direction)

将向量从局部坐标系转化全局坐标系。

  • Transform.InverseTransformDirection(Vector3 direction)

将向量从全局坐标系转换为局部坐标系。

  • Camera.ViewportToWorldPoint(Vector3 position)

将坐标点从视口坐标系转换为全局坐标系。

  • Camera.ScreenToViewportPoint(Vector3 position)

将坐标点从屏幕坐标系转换为视口坐标系。

  • Camera.ViewportToScreenPoint(Vector3 position)

将坐标点从视口坐标系转换为屏幕坐标系。

  • Camera.WorldToScreenPoint(Vector3 position)

将坐标点从全局坐标系转换为屏幕坐标系。

第十九章：脚本

必然事件

名称 触发条件 用途
Awake 脚本实例被创建时调用 用于游戏对象的初始化，该函数执行早于所有脚本的Start
Start Update函数第一次运行之前调用 用于游戏对象的初始化
Update 每帧调用一次 用于更新游戏场景和状态
FixedUpdate 每个固定物理时间间隔调用一次 用于更新物理状态
LateUpdate 每帧调用一次，在Update调用之后 更新游戏场景和状态，特别是与相机有关的更新

注意：

由于帧与帧之间可能会形成卡顿，因此严密的物理状态更新应该放在FixedUpdate()中，它保证了每个固定时间的调用。

Transform组件

Transform组件控制游戏对象在Unity场景中的位置、旋转和缩放，每个游戏对象都包含一个Transform组件。

常用的API与属性如下：

属性/API 说明
position 在世界坐标系中的位置
rotation 世界坐标系中以四元数表示的旋转
right 对象在世界坐标系中的右方向
Translate 按指定的方向和距离平移
Rotate 按指定的欧拉角旋转（相对于自身坐标系）
RotateAround 按给定的旋转轴和旋转角度进行旋转（相对于世界坐标系）
LookAt 旋转使得自身的前方向指向目标的位置

Time类

通过Time类来获取和时间有关的信息。

常用的属性如下：

属性 说明
time 只读：游戏从开始到现在经历的时间（秒）
deltaTime 只读：上一帧耗费的时间（秒）
fixedDeltaTime 物理引擎和FixedUpdate()的更新时间间隔

Coroutine：协程

  • 定义

和多线程类似、与主程序并行运行的程序。

  • 用途

用以实现让一段程序等待一段时间后继续运行的效果。

用更普通的话来说，就是：

注册一个每帧都连贯执行的函数，且每次都到yield处暂停处理。

  • API

常用的API如下：

API 说明
StartCoroutine 启动一个协同程序
StopCoroutine 终止一个协同程序
StopAllCoroutine 终止所有的协同程序
WaitForSeconds 等待数秒
WaitForFixedUpdate 等待直到下一次FixedUpdate调用
  • 注意

在C#中，协同程序的返回类型必须为IEnumerator，返回时必须要写yield return。

第二十五章：脚本优化

  • 在使用FixedUpdate()函数时，在方法内不要写太多无须重复调用的代码。
  • 删除没有使用的Update()函数，将耗时操作放在start()函数中。
  • 在类中、函数外定义游戏对象的组件与逻辑，函数中直接引用即可。
  • 当一个程序没必要每帧都执行时，可以使用Coroutines，定时重复调用使用InvokeRepeating()函数。
  • 尽量减少使用临时变量。
  • 优化数学计算，尽量用整型、用乘法代替除法、少用复杂数学函数。
