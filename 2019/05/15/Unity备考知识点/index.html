Unity备考知识点

发表于 2019-05-15 | 更新于 2019-05-16 | 分类于 -备考

嗯，选了这门课就要好好准备考试啊。

第一章：Unity简介

特点

Unity引擎的最主要特点是：一次开发就可以部署到所有主流的游戏平台。

面板与视图

  • Hierarchy：层级

Hierarchy视图用于显示当前游戏场景中的每个游戏对象。

  • Inspector：检视

Inspector视图用于显示游戏场景中，当前所选择游戏对象的详细信息和属性设置。包括对象的名称、标签、位置坐标、旋转角度、缩放、组件等信息。

  • Animator：动画控制器

Animator视图可以预览和设置角色行为。

  • Navigation：导航寻路

Navigation视图用于设置游戏角色的自动寻路。

第九章：游戏对象、组件和Prefabs

访问游戏对象

  • 通过名称来查找

使用函数GameObject.find(str name)，如果场景中存在制定名称的游戏对象，那么返回该对象的引用，否则返回null。

如果有多个重名的对象，则返回第一个对象。

  • 通过标签查找

使用函数GameObject.findWithTag(str tag)或GameObject.findGameObjectsWithTag(str tag)来获取指定标签游戏对象的第一个或全部，否则返回null。

组件

任何游戏对象都由组件构成，组件是实现一切功能所必需的，不同的组件有不同的功能。

常用的组件如下：

  • Mesh组件

Mesh：网格组件，如果不为网格添加一个Mesh Renderer组件，则该网格不会被渲染出来。

  • Physics组件

Physics：物理组件，用于模拟真实的物理行为。

  • Scripts组件

Scripts：脚本组件是必不可少的组成部分，它用于添加到游戏对象身上以实现各种交互操纵或其它功能。

组件的访问

  • 对于系统内置的常用组件，只需要直接在脚本里直接访问组件对应的成员变量即可。
组件名称 变量名 组件作用
Transform transform 对象位置、旋转、缩放
Rigidbody rigidbody 物理引擎的刚体属性
Renderer renderer 渲染物体模型
  • 如果访问的不是常用组件，或者欲访问脚本组件：

使用GetComponent<name>()或GetComponents<name>()得到对象的组件或组件列表。

调用该类函数比较耗时，应放在start()中。

Prefabs：预设体

  • 定义

一个游戏对象及其组件的集合与抽象。

  • 优点

使游戏对象及资源能够被重复利用，节约资源、提高开发的效率。

  • 缺点

对预设体进行更改，实例也将同步修改。

预设体的实例化

可通过静态、动态两种方式完成预设体的实例化。

  • 静态

将预设体拖动到Scene视图，或者Hierarchy视图中，便完成了一个预设体的实例化。

  • 动态

即通过脚本的方式进行预设体实例化，核心函数Instantiate有5个重载，常用的函数签名为Instantiate(Object original, Vector3 position, Quaternion rotation)。

函数的三个参数分别为：用来做复制的源对象original、实例对象的位置坐标position、实例对象的四元数旋转坐标rotation。

第十一章：动画系统

Humanoid

Humanoid：人形，通过骨骼映射，去识别其它动画，使之能应用到本身模型上。

  • 可以使用动画重定向功能：即一个人物模型的动画，可以供多个共同骨骼的模型使用。

  • 只通过动画便可以控制人物模型运动。

Avatar

使用Avatar来实现动画的重定向，即动画的复用。

在Rig标签页选择Humanoid后，Avatar Definition常用两种选项

  • Create From This Model：由本模型创建一个Avatar，供自己与其它相同Humanoid的骨骼来共同使用。

  • Copy From Other Avatar：使用其它模型的Avatar。

动画状态机

create一个Animation Controller，将已经制作好的各个动画片段拖入状态机中，形成一个个状态，通过Make Transition来设置状态变换的条件。

最后，将该状态机拖到人物模型身上即可。

第十二章：物理系统

Rigidbody：刚体组件

为游戏对象添加Rigidbody组件，实现该对象在场景当中的物理交互。

Rigidbody组件的常用参数如下：

  • Mass：用于设置游戏对象的质量。
  • Drag：设置游戏对象受力运动时的空气阻力。
  • Use Gravity：游戏对象是否使用重力。

Collider：碰撞体组件

选中一个游戏对象，为其添加不同类型的physics组件。

碰撞体组件的常用参数如下：

  • Is Trigger：触发器，选中该项则该碰撞体会忽略物理碰撞，用于触发碰撞事件。

碰撞检测的条件

两个游戏物体都有Collider组件，主动碰撞的游戏物体必须有Rigidbody组件（涉及到Character Controller组件）。

导航网格寻路步骤

  1. 选中可行走平面，在static选项下选中Navigation Static。
