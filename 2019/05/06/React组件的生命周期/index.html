React组件的生命周期

发表于 2019-05-06 | 更新于 2019-05-07 | 分类于 -前端框架

是时候停止使用componentWillMount()了。

组件的生命周期

对一个React组件进行剖析，可以将其生命周期划分为这几个阶段：

  1. 组件挂载
  2. 组件更新
  3. 组件卸载
  4. 异常处理

想要速览常用的生命周期函数，请查看官方生命周期速查表。

下述介绍中，常用的生命周期函数将会被加粗显示，而各函数的详细解释将被放在本文章的最后介绍。

组件挂载

组件挂载阶段调用的生命周期函数顺序如下：

  1. constructor()
  2. static getDerivedStateFromProps()
  3. render()
  4. componentDidMount()

注意：

componentWillMount()方法即将过时，在新代码中应该避免使用。

组件更新

当组件的props或state发生变化时会触发更新，这时调用的生命周期函数顺序如下：

  1. static getDerivedStateFromProps()
  2. shouldComponentUpdate()
  3. render()
  4. getSnapshotBeforeUpdate()
  5. componentDidUpate()

注意：

componentWillReceiveProps()方法即将过时，在新代码中应该避免使用。

当组件的渲染过程、生命周期函数、子组件的构造函数中抛出错误时，会按照顺序调用下述生命周期函数：

  1. static getDerivedStateFromError()
  2. componentDidCatch()

常用函数详解

constructor(props)

如果不需要初始化state或不为组件添加事件处理函数，则无需实现构造函数。

为解决this.props未定义的BUG，应该首先调用super(props)。

在构造函数中，你应该做的只有两件事：

  1. 给this.state赋值以初始化state
  2. 为组件的事件处理函数绑定this

也就是说，下述操作是被禁止的：

  • 在构造函数中调用this.setState()方法：

render()

render()方法是class组件中唯一必须实现的方法。

render()被调用时，会检查this.props和this.state的变化，并返回以下类型之一：

  • React元素：使用JSX语法描述的DOM节点或自定义组件，比如<div />和<myComponent />均为React元素。
  • 数组或fragments：即包含了多个React元素的数组或包含了多个兄弟节点的组件。
  • Portals：可以渲染子节点到不同的DOM子树中。
  • 字符串或数值类型：会被渲染为文本节点。
  • 布尔类型或null：什么都不渲染。

render()应该是个纯函数，它不应该与浏览器进行交互，换句话说，UI=render(data)。

注意：

如果shouldComponentUpdate()返回false，则不会调用render()。

componentDidMount()

该函数会在组件完成挂载，即渲染并插入到DOM树，之后调用。

一般我们将通过网络请求数据的操作放在这里进行。

在该函数中，你可以使用this.setState()，但不推荐，因为会导致性能问题。如果你的渲染依赖于DOM节点，使用这个方式也未尝不可。

shouldComponentUpdate(nextProps, nextState)

该函数被称为优化性能的关键——如果一个组件没有渲染的必要，那就干脆不渲染。

默认情况是state每次发生变化时都会重新渲染，该函数的返回值默认为true。在组件的首次渲染或使用forceUpate()时不会调用该方法。

通常都是通过手动比较this.props与nextProps、this.state与nextState，返回false告诉React跳过对该组件的更新。

componentDidUpate(prevProps, prevState, snapshot)

该函数会在更新后立即调用，首次渲染不会执行该方法。

同componentDidMount()一样，此时可以安全地使用DOM节点。

在该函数中，你可以使用this.setState()，但你必须保证该语句被包裹在一个条件语句中——总不能让它无限循环吧。

componentWillUnmount()

该函数会在组件卸载及销毁之前调用。一些垃圾处理应该放在该函数中进行。

在该函数中不应使用this.setState()——卸载组件后，永远不会再渲染它。
