利用JavaScript实现刷课

发表于 2019-01-28 | 更新于 2019-01-30 | 分类于 -JavaScript

我认为最愚蠢的事情莫过于——低效率的重复作业。这些东西就应该从根本上被消灭。

目的

前段时间，在应付考试复习闲暇，忽然想起了学院颁给我们的任务：在微信平台上参与一个新生安全教育、国防教育的学习。

当时碍于时间有限，便没有将其放在心上，姑且大致看了一两课的内容便去准备其它科目的考试了。之后决定完成这项任务的，便是听闻其他同学花了些钱请人帮忙代刷。当时我并没有动这种心思，反而有些轻视那些走捷径的同学，虽然人家只不过是合理消费罢了。

或许是这个刷新——扫一眼——划到底——刷新的重复劳动实在让人厌烦，周围不少同学都乖乖掏了钱。

嗯，有点兴趣。

想起了Atwood定律的我决定自己试一试。

摸索

动了这种念头之后，我便开始着手建立思路。在我漫无目的地进行思考时，有幸看到了一个回答，好吧，只不过是向服务器中特定的后台脚本发送请求而已。

所以我们需要做的就是，通过观察网址、查看网页源代码、以“持续日志”记录网络请求，得到我们想要的后台脚本地址、请求发送参数。

可行性研究

倘若你跟我说，让我写个脚本实现盈利性操作，我绝对会以看猴子的眼神去看你——是否违反法律暂且不表，网站的安全性、复杂度能超乎你的想象。随便让我这种外行人举一个例子：每次请求均生成一个带有时效性的token，就能除去绝大部分非法请求。

得益于该学习平台以公众号形式而不是小程序形式（暂不支持网页打开）进行投放，以及平台欠缺安全性防卫措施，就给了我这种投机分子一定的机会。

经过多次尝试之后，不怎么费力就找到了我感兴趣的东西。

获取课程列表的请求：

http://example.com/pharos/usercourse/listCourse.do?timestamp=1545534138

获取课程完成的允许：

http://example.com/pharos/usercourse/study.do?timestamp=1545534138

每一课完成之后的请求：

http://example.com/pharos/usercourse/getCourseUrl.do?timestamp=1545534138

http://example.com/pharos/usercourse/finish.do

接下来，只要按照顺序，提供所需要的参数，进行请求即可。

其实，最关键的思路不外乎流程、请求、参数这三方面。
